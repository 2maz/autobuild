Autobuild::Configurable

Base class for packages that require a configuration + build step.

Child classes must provide a # file which represents the last configuration step done. This file is updated by a call to # (see below)

Three new methods are added, which can be reimplemented in child classes:

  • configure does configure the package. It is ran after all depended-upon packages are installed.

  • build is ran after configure if the configure stamp and/or the source files have been updated. The # stampfile represents when the last build has been done. The build must be done in the # directory.

  • install is ran after build.
